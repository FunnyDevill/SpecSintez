СпецСинтез – Административная панель и публичный сайт

Полнофункциональный веб-сайт компании «НПО СпецСинтез» с административной панелью. Публичная часть представляет собой каталог продукции, новости, отраслевые страницы и форму обратной связи. Административная панель позволяет управлять товарами, категориями, новостями, типами упаковок, загружать изображения, использовать WYSIWYG-редактор.

Стек: Node.js, Express 5, PostgreSQL, EJS, Multer, Nodemailer, Quill, Helmet, BCrypt, Mocha/Chai.

2. Установка и запуск
Требования

    Node.js 18+

    PostgreSQL 12+

    npm

Установка

    Клонируйте репозиторий или распакуйте архив.

    Создайте базу данных specsintez (и при необходимости specsintez_test для тестов).

    Скопируйте .env.example в .env и заполните его (см. пример ниже).

    Выполните npm install.

    Запустите сервер: npm start или node server.js.
    Для production рекомендуется использовать PM2: pm2 start server.js --name specsintez.

Пример .env
env

# База данных
DB_USER=postgres
DB_HOST=localhost
DB_NAME=specsintez
DB_PASSWORD=postgres
DB_PORT=5432

# Тестовая база
TEST_DATABASE_URL=postgres://postgres:postgres@localhost:5432/specsintez_test

# Сессия
SESSION_SECRET=your-random-secret

# Администратор
ADMIN_USER=admin
ADMIN_PASS=$2b$10$...   # хеш пароля, полученный через bcrypt

# SMTP для отправки заявок
SMTP_HOST=smtp.mail.ru
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=artem-korytov@mail.ru
SMTP_PASS=пароль-приложения-mailru

# Порт
PORT=5500

Запуск тестов
bash

npm test

Тесты используют отдельную базу данных, указанную в TEST_DATABASE_URL. Перед запуском убедитесь, что она создана. Подробнее в разделе «Тестирование».
3. Публичная часть

    Главная (/) – видео-фон, отрасли, карточки товаров и новостей (динамическая загрузка через API).

    Каталог (/catalog.html) – боковое дерево категорий, постраничный вывод товаров.

    Детальная товара (/product.html?slug=...) – галерея, описание, отрасли, типы упаковок.

    Новости (/news.html) – hero-новость и список.

    Детальная новости (/news-detail.html?slug=...).

    Статические страницы: О компании, Партнёрам, Производство ПАВ, Контрактное производство.

    Формы обратной связи: на странице «Контрактное производство» и в CTA-блоке футера. Данные отправляются на email через Nodemailer (настроен на artem-korytov@mail.ru).

Публичные API
Метод	URL	Описание
GET	/api/news	Hero-новость + список из 20 новостей
GET	/api/news/:slug	Одна новость по slug
GET	/api/categories?all=true	Категории (активные/все)
GET	/api/products?category_id=X&all=true	Товары с фильтром
GET	/api/random-products?limit=4	Случайные товары
GET	/api/products/:id	Товар по ID
GET	/api/products-by-slug/:slug	Полные данные товара (категории, упаковки, изображения)
4. Административная панель

Доступ после авторизации: /admin
Логин и пароль задаются в .env.
Функциональность

    Товары: создание, редактирование, удаление. Поиск по названию/slug, сортировка, пагинация. Полное описание через Quill-редактор. Загрузка изображений (drag-and-drop) с возможностью выбора главного.

    Категории: древовидная структура с поддержкой вложенности. Загрузка SVG-иконок только для корневых категорий.

    Новости: Quill для контента, загрузка изображения, дата публикации, SEO-поля.

    Типы упаковок: загрузка иконок, привязка к товарам с указанием объёма.

    Безопасность: CSRF-защита, Content Security Policy с nonce, сессии.

Основные маршруты админки
Метод	URL	Действие
GET/POST	/admin/products	Список товаров
GET/POST	/admin/products/create	Создание товара
GET/POST	/admin/products/:id/edit	Редактирование
POST	/admin/products/:id/delete	Удаление
AJAX	/admin/products/:id/images/...	Загрузка/удаление/установка главного изображения
...	/admin/categories/...	Аналогично для категорий
...	/admin/news/...	Аналогично для новостей
...	/admin/package-types/...	Управление типами упаковок
GET/POST	/admin/products/:id/package-types	Привязка упаковок
5. Безопасность

    Helmet устанавливает защитные HTTP-заголовки (CSP, HSTS, X-Frame-Options и др.).

    CSP с nonce: все встроенные скрипты подписываются уникальным nonce, генерируемым для каждого запроса. Это исключает выполнение вредоносного кода.

    CSRF-защита: все формы и AJAX-запросы содержат токен, проверяемый middleware csurf.

    Сессии с флагом httpOnly.

    CORS разрешает запросы только с доверенных доменов (рекомендуется настроить перед продакшеном).

    Nodemailer использует пароль приложения, а не основной пароль почты.

6. Тестирование

Тесты охватывают CRUD-операции админки и публичные API.
Инфраструктура: Mocha, Chai, Supertest.
Запуск
bash

npm test

Структура тестов

    test/setup.js – создание и очистка таблиц.

    test/products.test.js – товары.

    test/news.test.js – новости.

    test/categories.test.js – категории.

    test/package-types.test.js – типы упаковок.

    test/public-api.test.js – публичные API.

Тестовая база должна существовать (specsintez_test), параметры подключения задаются через TEST_DATABASE_URL. В тестовой среде отключены аутентификация и CSRF.
7. Деплой и передача заказчику
Локальный запуск
bash

npm install
# заполнить .env
node server.js

Production (VPS)

    Используйте PM2: pm2 start server.js --name specsintez, pm2 save, pm2 startup.

    Настройте Nginx в качестве обратного прокси на порт 5500.

    Ограничьте CORS в server.js.


Сборка Docker (опционально)

Прилагаются Dockerfile и docker-compose.yml (см. корень проекта).
Запуск: docker-compose up -d.
